# 搜索准确性与有效性优化测试报告

- 日期：2026-07-18
- 测试环境：macOS、本地 Vite 前端、FastAPI/Uvicorn 后端、SQLite
- 前端地址：http://127.0.0.1:5173/
- 后端地址：http://127.0.0.1:8001/

## 一、问题根因

1. 旧版 360/搜狗/Bing 解析会从一个搜索摘要块中提取多个网盘链接，再把同一个摘要标题赋给多个链接，导致“标题和链接张冠李戴”。
2. 旧版仅使用正则检查链接格式，没有访问网盘的公开接口确认分享是否已被取消、过期、违规或删除。
3. 复制后使用脚本弹窗打开页面，会被部分浏览器的弹窗策略拦截。

## 二、参考项目与设计依据

参考 GitHub 开源项目：[fish2018/pansou](https://github.com/fish2018/pansou)。

采用了其中两项经过实践验证的设计思想：

1. 每个网盘链接保存独立作品标题，不用整段消息的总标题覆盖所有链接。
2. 按网盘类型调用公开接口检查链接，区分有效、失效、需要提取码和无法确认。

本项目没有整体复制 PanSou，而是新增专用适配源，并使用 Python/FastAPI 实现本地严格校验层。

## 三、完成的优化

### 1. 搜索源

- 新增 PanSou 结构化聚合源，使用每条链接自己的 `note` 标题。
- PanSou 使用 POST 搜索接口，减少公共节点 GET 链路延迟。
- 搜狗/Bing 保留为兜底，但启用“一卡一链接”严格解析。
- 360 默认关闭；解析器也已改成只处理真实结果卡片，并拒绝一卡多链接。

### 2. 有效性检测

当前严格检测支持：

- 夸克网盘：调用分享 token 接口，识别失效、缺码和有效资源，并读取真实资源标题及文件数量。
- 百度网盘：验证提取码，再调用分享列表接口，识别取消、失效、错误提取码，并读取真实资源标题。
- 阿里云盘：调用匿名分享信息接口，检查分享状态、文件数量和标题。
- 123 云盘：调用分享信息接口确认分享状态。

缺少稳定公开检测接口的网盘不会仅因为页面返回 HTTP 200 就被标成有效；严格模式会暂不展示，避免误报。

### 3. 准确性检测

- 中文关键词必须按原顺序出现，允许中间插入“资源”等词，但不允许关键词字符在长摘要中无序散落。
- 过滤网赚、拉新、收益、广告域名等明显垃圾结果。
- 非结构化搜索源必须同时通过“搜索标题—网盘真实标题”一致性检查。
- 结构化来源保留原作品标题，同时在页面展示网盘接口返回的“网盘实际内容”，方便用户交叉确认。
- 一个搜索卡片内存在多个不同链接时直接丢弃，避免猜测标题归属。

### 4. 缓存可靠性

- 搜索缓存升级为 `v6-strict-verified`，旧错误缓存自动隔离。
- 不再用空结果覆盖历史有效缓存。
- 上游临时超时时，可重新检测历史有效快照后降级返回。
- 搜索缓存命中时仍经过链接检测层；检测状态只短缓存两分钟，到期后重新向网盘确认。

### 5. 复制并打开

- “复制链接”使用原生 `<a target="_blank" rel="noopener noreferrer">`，复制成功后新建页签/窗口打开该网盘页面。
- “复制链接及提取码”使用同样的新页签/窗口行为。
- 原搜索页保持不动，用户可以继续查看和复制其他结果。

## 六、真实接口测试

### 关键词：电影分享

上游候选中包含：

- 已失效夸克链接：被过滤。
- 已取消百度链接：被过滤。
- 网赚/拉新广告资源：被过滤。
- “疯狂动物城2”夸克链接：网盘确认有效，保留。
- “疯狂动物城2”百度链接：提取码验证成功、文件列表存在，保留。

最终返回 2 条严格有效结果，全部带 `validation_status=valid` 和网盘实际资源标题。

### 关键词：琅琊榜

由于上游索引会实时变化，多轮测试返回 3～31 条严格有效结果，涵盖百度、夸克、阿里云盘；最终一轮返回 31 条，页面首屏标题和网盘实际内容均包含“琅琊榜”或“琅琊榜之风起长林”。

首次搜索约 5.85 秒，包含多源检索和网盘有效性检查；缓存命中 HTTP 响应约 8ms，接口字段 `search_time_ms=0`。

## 七、浏览器端到端测试

1. 浏览器输入“电影分享”并回车，显示 2 条结果。
2. 两条结果都显示“✓ 链接有效”。
3. 两条结果都显示“网盘实际内容”。
4. 点击百度结果“复制链接”后，通过新页签/窗口进入百度网盘，原搜索页保持不动，页面标题显示对应的“疯狂动物城2”资源。
5. 点击夸克结果“复制链接及提取码”后，通过新页签/窗口进入夸克网盘。
6. 夸克落地页实际显示：共 7 个文件、63.3G、永久有效，文件夹名称为“疯KD物城”。
7. 浏览器再次搜索“琅琊榜”，显示全部经过验证的结果，页面标题和网盘实际内容对应；最终接口复测返回 31 条有效结果。
8. 浏览器点击“夸克网盘”筛选，返回 2 条结果，结果卡片全部为夸克且全部显示“链接有效”。
9. 浏览器点击“百度网盘”筛选，返回 1 条结果，结果卡片全部为百度且全部显示“链接有效”。

## 八、验收结论

本轮修复已经把默认策略从“尽量多抓链接”改为“宁缺毋滥，只展示当前能被网盘确认的链接”。

有效性的准确含义是：结果返回时，后端刚刚通过对应网盘公开接口确认分享存在；任何外部链接仍可能在之后被分享者主动取消，因此系统会在短检测缓存到期后重新验证，而不是永久相信旧状态。
